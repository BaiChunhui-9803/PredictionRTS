# PredictionRTS 项目总结报告

## 一、项目概述

### 1.1 项目定位

PredictionRTS 是一个面向**即时战略游戏(RTS)智能体行为预测与决策**的深度学习研究项目。该项目基于 StarCraft II (PySC2) 环境，通过分层 Q-Learning 智能体采集对战数据，并利用 Transformer 架构进行状态序列预测和决策建模。

### 1.2 核心功能

| 功能模块 | 描述 |
|---------|------|
| 数据采集 | 在 PySC2 环境中运行 SmartAgent，采集状态/动作/奖励序列 |
| 状态序列预测 | 使用 Transformer 模型预测未来状态序列 |
| 决策预测 | 使用 Decision Transformer 预测最优动作 |
| 结构分析 | BK-Tree 状态聚类、状态距离计算、DTW 轨迹距离 |
| 可视化 | Fitness Landscape、State Landscape、预测结果可视化 |

### 1.3 研究动机

- **问题**: RTS 游戏状态高维、离散、带时序与空间结构，直接端到端学习困难
- **思路**: 
  - 用 BK-Tree + 聚类将原始状态离散化为状态节点
  - 用自定义状态距离刻画状态相似性
  - 用 Transformer 进行序列建模，引入距离诱导的表征
- **目标**: 学习状态演化与状态→动作的映射，为 RTS 轨迹预测和策略分析提供可解释模型

## 三、核心算法

### 3.1 模型架构

#### TrajectoryTransformer
- **文件**: `models/Transformer.py`
- **结构**: Embedding + 可学习位置编码 + TransformerEncoder
- **输出**: Linear(d_model, vocab_size) 进行下一 token 预测
- **特点**: 因果 mask，支持 padding mask

#### SimilarityGuidedTransformer (SGTransformer)
- **文件**: `models/SGTransformer.py`
- **创新**: 在注意力机制中引入外部 `sim_bias`（状态距离诱导的偏置）
- **特点**: 可学习的 `bias_scale` 参数缩放相似度偏置

#### DecisionTransformer
- **文件**: `models/DecisionTransformer.py`
- **输入**: RTG (Return-to-Go) + State + Action 三元组
- **结构**: 按 [R1,S1,A1, R2,S2,A2,...] 交织排列
- **输出**: 取 state 位置 (1,4,7,...) 的输出预测 action logits

### 3.2 状态距离与 BK-Tree

| 组件 | 说明 |
|------|------|
| **BK-Tree** | 按聚类/状态组织的树结构，支持高效最近邻查询 |
| **custom_distance** | 多维距离计算（坐标+生命值），支持匈牙利匹配 |
| **DTW** | 基于状态距离矩阵的动态时间规整，用于轨迹间距离 |

### 3.3 距离诱导的表征

1. **MDS 初始化**: 用状态距离矩阵做 MDS 降维，初始化 embedding
2. **SG 注意力偏置**: 在注意力中引入相似度偏置
3. **空间先验**: 预测时用状态距离构造高斯先验，偏向几何近邻状态

## 七、创新点与学术定位

### 7.1 方法创新

1. **状态离散化与几何结构**
   - BK-Tree + 聚类将 RTS 状态映射为离散 ID
   - 自定义状态距离（空间+生命值）刻画状态相似性

2. **距离诱导的 Transformer 表征**
   - MDS 初始化：使 embedding 空间与状态距离一致
   - SGTransformer：注意力受状态相似度引导
   - 空间先验：自回归预测时偏向几何近邻

3. **RTS 场景下的 Decision Transformer**
   - 状态=离散状态 ID，动作=离散动作表
   - RTG 由每步奖励反向累加
   - MDS 初始化 state embedding

4. **Fitness/State Landscape 可视化**
   - 轨迹 DTW 距离 + 游戏结果 → Fitness Landscape
   - 状态距离矩阵 + 状态价值 → State Landscape

### 7.2 学术定位

- **领域**: RTS AI / 游戏 AI + 离线强化学习 / 序列建模
- **相关工作**: Trajectory Transformer, Decision Transformer, RTS 状态抽象
- **潜在贡献**: 距离感知的序列建模、RTS 专用状态离散化

### 7.3 可改进方向

1. **参数管理**: 引入 argparse 或 Hydra + YAML
2. **实验记录**: 对 params 与 suffix 做日志持久化
3. **对比基线**: 与 Vanilla Transformer 及其他离线 RL 方法对比
