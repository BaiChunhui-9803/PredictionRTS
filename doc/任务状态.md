## 1. 项目概述

交互式知识图谱可视化网页工具（Streamlit + pyvis），用于浏览 KG、链式推理预测、滚动推演和原始数据查询。

### 技术栈
- **Python 3.8.10** + **numpy 1.24.4**（必须使用 `.venv`，系统 Python numpy 2.x 不兼容 pkl）
- **Streamlit 1.40.1**
- **pyvis**（力导向图渲染）
- **Plotly**（趋势图）
- **NetworkX**（图数据结构）

### 关键文件

| 文件 | 职责 |
|------|------|
| `scripts/visualize_kg_web.py` (~2985 行) | 主 UI 文件，所有 Tab 的渲染逻辑 |
| `src/decision/kg_beam_search.py` (~369 行) | Beam Search 引擎，`find_optimal_action()`, `get_beam_paths()`, `BeamSearchResult` |

## 4. 关键数据流

### 链式推理 Tab 数据流
```
find_optimal_action(kg, transitions, state)
  → returns (action, info)
  → info["all_results"]: List[BeamSearchResult]  # 扁平列表，每项有 parent_idx
  → get_beam_paths(results)  # 从叶节点回溯 parent_idx → List[List[BeamSearchResult]]
  → _compute_path_metrics(path)  # 每条路径的多维指标
  → 综合评分 + 排序 → 路径推荐表格
  → 选中路径 → highlight_indices
  → render_beam_tree(results, highlight_indices=highlight_set)  # 用 parent_idx 构建树 + 高亮
  → match_beam_paths(paths_dict, ...)  # 片段匹配
  → 路径详情表格  # 按路径索引 + 分叉标记
```

### 滚动推演 Tab 数据流
```
chain_rollout(kg, transitions, state, ...)
  → returns RolloutResult
    ├── nodes: Dict[int, RolloutNode]  # 统一树
    ├── chosen_path_ids: List[int]  # 主路径节点 ID
    ├── termination_reason: str
    └── beam_results_by_step: Dict[int, List[Dict]]  # 每步原始 beam_results
  → 主路径信息（摘要/趋势图/匹配/轨迹）
  → 束搜索追溯（从 beam_results_by_step 直接反序列化 → BeamSearchResult → render_beam_tree）
  → 完整推演树（从 nodes dict 渲染）
```

### BeamSearchResult 结构
```python
@dataclass
class BeamSearchResult:
    step: int
    state: int
    action: str
    cumulative_probability: float
    quality_score: float
    win_rate: float
    avg_step_reward: float
    avg_future_reward: float
    beam_id: int  # 仅在单步内有效，跨步无意义！
    parent_idx: Optional[int]  # 在 all_results 列表中的索引
```

### RolloutNode 结构
```python
@dataclass

    id: int

    avg_future_reward: float
    avg_step_reward: float
    visits: int
    transition_prob: float
    cumulative_probability: float
    rollout_depth: int  # -1 for root
    is_on_chosen_path: bool
    is_terminal: bool
    is_beam_root: bool
```

## 5. 已知问题 / 注意事项

1. **LSP 误报**：font_color 类型、Options.__setitem__、matplotlib 类型标注等均为类型检查误报，不影响运行
2. **numpy 版本**：必须用 `.venv\Scripts\python.exe`，系统 Python numpy 2.x 不兼容
3. **`@st.cache_data`**：修改缓存函数签名后需 Clear cache（新增了 rollout_mode 等参数，需清缓存）
4. **`__pycache__`**：修改 chain_rollout.py 后需清除 `src/decision/__pycache__` 并重启 Streamlit
5. **beam_id 跨步无意义**：beam search 每步重新编号，同一 beam_id 在不同 step 的节点之间无父子关系
6. **路径树图着色**：当前仍按 beam_id 着色（跨步无意义），计划改为按路径索引着色
7. **`st.segmented_control` multi 模式**：Streamlit 1.40.1 不支持 `selection_mode="fixed"`，使用默认 multi + 空选择防护
8. **render_beam_tree() highlight_indices**：用 `r in selected_path`（对象比较）构建 set，依赖 Python 对象同一性
9. **多步模式 beam_results_by_step**：多步模式下 key 是全局步骤号（非分段），每段规划只存一次 beam results
10. **`_cached_rollout_results` 签名变更**：新增 `rollout_mode`, `enable_backup`, `score_threshold`, `distance_threshold` 参数，首次运行需清缓存
