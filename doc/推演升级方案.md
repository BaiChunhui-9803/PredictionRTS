> 创建时间: 2026-04-09
> 状态: 执行中

## 概述

两个升级目标：
1. **束搜索追溯 UI 同步**：让滚动推演的束搜索追溯区域具备与链式推理一致的路径推荐表 + 路径详情 + highlight 联动
2. **多步推演模式**：`chain_rollout()` 支持多步推演（沿完整路径执行）+ 备选路径快速切换

#### 2. 子树路径解析

#### 3. 路径推荐表（新增）

复用 `_compute_path_metrics()` + 综合评分 + 排序，14 列 + 综合评分：
- `#`, `Path`, `长度`, `首步动作`, `首步目标`, `末状态`, `置信度`, `平均Quality`, `平均胜率`, `末Quality`, `末胜率`, `Q趋势`, `WR趋势`, `平均奖励`, `综合评分`
- `st.dataframe(on_select="rerun", selection_mode="single-row")`
- 综合评分公式：`0.30*conf + 0.30*qual + 0.30*wr + 0.10*reward`（min-max 归一化）

#### 4. 路径详情表（新增）

与链式推理一致：
- 按共享前缀排序
- `↑` 标记共享步骤，`➤` 标记分叉点
- 列：`Path`, `分叉`, `Step`, `State`, `Action`, `Cum Prob`, `Quality`, `Win Rate`, `Step Reward`, `Future Reward`

- 路径推荐表 `on_select="rerun"` + 死循环守卫（`new_path != old_path`）
- 选中路径 → `highlight_set = {i for i, r in enumerate(beam_results_list) if r in selected_path}`
- `render_beam_tree(beam_results_list, highlight_indices=highlight_set)`

#### 6. Beam 路径数据表适配

当前按节点展示的简单表替换为路径维度的信息。保留 Chosen 标记列（标识实际推演选中的动作/状态）。

#### 7. Session state key

- 链式推理：`session_state.pred_selected_path`
- 滚动推演：`session_state.roll_beam_selected_path`（避免冲突）

## 第二部分：多步推演模式

### 设计决策

| 问题 | 决策 |
|------|------|
| 多步路径选择策略 | 综合评分选路径（0.30*conf + 0.30*qual + 0.30*wr + 0.10*reward） |
| 切换点匹配方式 | 可配置阈值匹配：精确优先 + 距离回退 |
| 备选路径保留数量 | 动态裁剪：评分阈值初筛 + 分叉点去重 |
| 备选路径刷新策略 | 每轮刷新（re-search 时重新生成主路径+备选路径） |

### 数据结构

#### SwitchPoint

```python
@dataclass
class SwitchPoint:
    plan_position: int                    # 主路径中的步骤位置（0-based）
    predicted_state: int                  # 主路径在该步骤预测的下一状态
    backup_path_idx: int                  # 备选路径索引
    backup_step_in_path: int              # 备选路径中切换点的步骤位置
    remaining_actions: List[str]          # 切换后的剩余动作序列
    match_type: str                       # "exact" | "distance"
```

#### PlanSegment

```python
@dataclass
class PlanSegment:
    segment_type: str                     # "initial_plan" | "backup_switch" | "re_search"
    source_path_idx: int                  # 使用的路径索引（主路径=0）
    actions_planned: List[str]            # 该段规划的动作序列
    divergence_step: int                  # 在第几步发生了偏移（-1=正常走完）
    divergence_type: str                  # "backup_switch" | "re_search" | "none"
```

#### RolloutResult 扩展字段

```python
rollout_mode: str = "single_step"                          # "single_step" | "multi_step"
plan_segments: List[Dict] = field(default_factory=list)     # 每段的执行记录
total_re_searches: int = 0                                  # 总重新规划次数
total_backup_switches: int = 0                              # 总备选切换次数
switch_points_by_segment: Dict[int, List[Dict]] = ...       # 每段的切换点
```

### chain_rollout() 核心改造

#### 单步模式（`rollout_mode="single_step"`）

保持现有逻辑不变。

#### 多步模式（`rollout_mode="multi_step"`）

```
初始化：
  current_state = start_state
  segment_count = 0

循环 while segment_count < max_rollout_steps:

  ── Phase 1: 规划 ──
  action, info = find_optimal_action(kg, transitions, current_state,
                                      beam_width, lookahead_steps, ...)
  results = info["all_results"]
  beam_paths = get_beam_paths(results)

  ── Phase 2: 选主路径 ──
  main_path = 选综合评分最高的路径（排除根节点后的动作序列）
  backup_paths = 动态裁剪后的备选路径

  ── Phase 3: 构建切换点 ──
  switching_map = build_switching_map(main_path, backup_paths, enable_backup)

  ── Phase 4: 沿主路径执行 ──
  segment = PlanSegment(segment_type="initial_plan"|"re_search", ...)

  for plan_step in range(len(main_path) - 1):  # -1 因为根是当前状态
    planned_action = main_path[plan_step + 1].action
    predicted_next = main_path[plan_step + 1].state

    actual_next = 执行转移(current_state, planned_action, transitions, rng, ...)

    if actual_next == predicted_next:
      继续：current_state = actual_next
    elif actual_next in switching_map:
      备选切换：切换到备选路径，更新 segment 记录
      current_state = actual_next
      沿备选路径剩余动作继续执行（同样检查切换）
      break
    elif enable_distance_matching:
      用距离矩阵找最近匹配点
      if 距离 < threshold:
        模糊切换
      else:
        重新规划：break，重新进入 Phase 1
    else:
      重新规划：break

  segment_count += 1
```

### build_switching_map 动态裁剪算法

```python
def build_switching_map(
    main_path: List[BeamSearchResult],
    all_paths: List[List[BeamSearchResult]],
    score_threshold: float = 0.3,
    max_per_fork: int = 3,
) -> Dict[int, List[SwitchPoint]]:
```

裁剪策略：
1. **评分阈值初筛**：计算每条路径的综合评分，丢弃评分 < `max_score * score_threshold` 的路径
2. **分叉点分析**：找到每条备选路径与主路径的分叉步骤 `fork_step`
3. **分叉点去重**：每个分叉步骤只保留评分最高的 `max_per_fork` 条路径
4. **构建切换点**：对保留的每条备选路径，从分叉步骤开始，每个步骤生成 `SwitchPoint`

### find_closest_switch_point 模糊匹配

```python
def find_closest_switch_point(
    actual_state: int,
    switching_points: List[SwitchPoint],
    dist_matrix: Optional[np.ndarray],
    distance_threshold: float,
) -> Optional[SwitchPoint]:
```

- 精确匹配优先：遍历 switching_points，若 `predicted_state == actual_state` 则返回
- 距离回退：若 dist_matrix 可用，找距离最小且 < threshold 的点

### Sidebar 新增参数

| 参数 | UI 控件 | 条件显示 | 默认值 |
|------|---------|----------|--------|
| 推演模式 | `st.segmented_control(["单步推演", "多步推演"])` | 始终 | "单步推演" |

### 推演轨迹表格增强

新增列：
- **推演段**：第几段规划（segment index）
- **段类型**：初始规划 / 备选切换 / 重新规划
- **路径编号**：当前执行的是哪条 beam path

推演摘要新增指标：
- **总规划次数**（re-search 次数）
- **备选切换次数**
- **主路径执行步数 / 总步数**（路径命中率）

### 束搜索追溯适配多步模式

- `single_step` 模式：每步一次 beam search → 现有 `beam_results_by_step[step]` 结构不变
- `multi_step` 模式：每段一次 beam search → `beam_results_by_step` 改为 `{segment_idx: {..., "plan_step": int, "beam_results": [...]}}`
