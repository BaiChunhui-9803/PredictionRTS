> 文档版本：v1.0  
> 创建日期：2026-04-09  
> 状态：待执行

## 1. 背景与问题诊断

### 当前架构

滚动推演（Tab "🎮 滚动推演"）每步调用 `find_optimal_action()` 进行束搜索，从搜索结果中选取一个动作执行，然后推进到下一个状态。

### 根本缺陷

**缺陷 1：信息丢弃**

每步束搜索返回完整的 `List[BeamSearchResult]`（beam_width × lookahead_steps 条路径树），但 `_rank_candidate_actions()` 只提取各 beam 的**第一步动作**，整个前瞻路径全部丢弃。

```
束搜索返回：
  S0 →(a1)→ S1 →(a2)→ S3 →(a3)→ S5     ← beam 0, 完整路径
  S0 →(a4)→ S2 →(a5)→ S6 →(a6)→ S8     ← beam 1, 完整路径
  S0 →(a7)→ S4 →(a8)→ S7 →(a9)→ S9     ← beam 2, 完整路径

当前只用了：[a1, a4, a7]  ← 三个首步动作
丢弃了：S3, S5, S6, S8, S7, S9 及其全部质量信息
```

**缺陷 2：动作选择过于简单**

`_rank_candidate_actions()` 排序依据仅为 beam 排名或转移概率，无法利用前瞻路径上的多维信息（下游胜率、终端分布、路径多样性等）。

**缺陷 3：不可追溯**

推演结束后无法回溯每步的束搜索决策过程，无法排查"无可用动作"等提前终止的原因。

---

## 2. 目标

1. **统一树结构**：将所有步的束搜索结果合并为一棵完整的推演树（主路径 + 所有探索过的分支）
2. **丰富节点数据**：每个节点包含完整状态信息（quality, win_rate, future_reward, transition_prob 等）
3. **多维度动作选择**：利用子树信息做决策（子树平均胜率、最大 quality 等）
4. **全链追溯**：支持精确查询每步束搜索给出的完整路径树，含路径树图可视化

## 3. 核心数据结构

### 3.1 `RolloutNode`

```python
@dataclass
class RolloutNode:
    # --- 树结构 ---
    id: int                          # 全局唯一节点 ID
    parent_id: Optional[int]         # 父节点 ID（根节点为 None）
    children_ids: List[int]          # 子节点 ID 列表

    # --- 状态信息 ---
    state: int                       # 状态 ID
    action: Optional[str]            # 到达此状态的动作（根节点为 None）
    beam_id: Optional[int]           # 产生此节点的 beam 编号（根节点为 None）

    # --- 质量指标 ---
    quality_score: float
    win_rate: float
    avg_future_reward: float
    avg_step_reward: float
    visits: int                      # 该 state-action 对的访问次数

    # --- 概率信息 ---
    transition_prob: float           # P(this_state | parent_state, action)
    cumulative_probability: float    # 从根到此节点的累积概率

    # --- 标记 ---
    rollout_depth: int               # 在主路径上的深度（-1 表示非主路径节点）
    is_on_chosen_path: bool          # 是否在主推演路径上
    is_terminal: bool                # 是否终端状态
    is_beam_root: bool               # 是否是某步束搜索的根节点（主路径节点都是）

    # --- 子树聚合指标（惰性计算） ---
    subtree_max_quality: Optional[float] = None
    subtree_avg_win_rate: Optional[float] = None
    subtree_avg_future_reward: Optional[float] = None
    subtree_n_terminals: Optional[int] = None
```

### 3.2 `RolloutResult`

```python
@dataclass
class RolloutResult:
    nodes: Dict[int, RolloutNode]    # 所有节点（id → node）
    root_id: int                     # 根节点 ID
    chosen_path_ids: List[int]       # 主路径上的节点 ID 序列
    termination_reason: str          # "终端状态" | "达到最大推演步数" | "无可用动作" | "累积概率过低"

    def get_beam_subtree_ids(self, rollout_step: int) -> List[int]:
        """获取某步束搜索产生的所有节点 ID"""
        ...

    def get_chosen_node_at_step(self, rollout_step: int) -> RolloutNode:
        """获取主路径上某步的节点"""
        ...

    def compute_subtree_metrics(self, node_id: int) -> Dict:
        """计算并缓存子树的聚合指标"""
        ...

    def get_action_candidates_at_step(self, rollout_step: int) -> List[Dict]:
        """获取某步的所有候选动作及其子树指标"""
        ...
```

### 3.3 树的构建算法

```
初始化：
  nodes = {0: RolloutNode(id=0, state=start_state, rollout_depth=0,
                          is_beam_root=True, is_on_chosen_path=True)}
  next_id = 1
  chosen_path = [0]
  current_id = 0

每步（step = 0, 1, 2, ...）：
  1. current_node = nodes[current_id]
     current_state = current_node.state

  2. 如果 current_state 是终端 → 终止

  3. find_optimal_action(current_state) → beam_results
     如果无结果 → 终止

  4. 将 beam_results 转换为子树节点，挂载到 current_id 下：
     a. beam_results[0]（step=0, 根节点）→ 映射到 current_node（已存在）
     b. 对每个非根 BeamSearchResult r：
        - 查找其 parent_idx 对应的已映射节点 mapped_parent_id
        - 创建新 RolloutNode(id=next_id, parent_id=mapped_parent_id, ...)
        - 添加到 nodes 和 mapped_parent 的 children_ids

  5. 按动作选择策略，从 current_node 的子节点中选择一个动作
     - 按 action 分组子节点
     - 计算每组的子树聚合指标
     - 按策略排序，尝试每个动作的 _pick_next_state()

  6. 确定实际 next_state：
     a. 如果与 beam 预测一致 → 标记对应子节点 is_on_chosen_path=True
        current_id = 该子节点 ID
     b. 如果不一致 → 在 chosen action 下创建新节点
        current_id = 新节点 ID
        注意：该新节点会成为下一步束搜索的根

  7. chosen_path.append(current_id)
     nodes[current_id].rollout_depth = step + 1
     nodes[current_id].is_beam_root = True

终止：
  termination_reason = ...
```

### 3.4 next_state 不一致时的处理

**核心原则**：next_state 没有按照 beam 预测的来，说明实际环境偏离了预测，原有分支不再适用，需要在新的 next_state 上重新执行束搜索。

**具体处理**：
- beam 预测的子树节点保留在树中（标记为探索分支，`is_on_chosen_path=False`），用于追溯分析
- 在 chosen action 下创建新的实际 next_state 节点（`is_on_chosen_path=True, is_beam_root=True`）
- 下一步从该新节点开始新一轮束搜索

## 4. 动作选择策略（6 种）

所有策略共享同一个框架：按 action 分组子节点 → 计算子树指标 → 按策略排序 → 逐个尝试 `_pick_next_state()`。

| 策略 | 排序依据 | 说明 |
|---|---|---|
| `best_beam` | beam_rank（beam 0 > beam 1 > ...） | 默认策略，信任束搜索排名 |
| `best_subtree_quality` | 子树最大 quality_score | 选子树中最优状态质量最高的动作 |
| `best_subtree_winrate` | 子树平均 win_rate | 选子树平均胜率最高的动作 |
| `highest_transition_prob` | 动作的转移总计数 | 选转移计数最高的动作 |
| `random_beam` | 随机打乱 | 从所有候选中随机选 |
| `epsilon_greedy` | 1-ε 按 beam_rank，ε 随机 | ε-贪心探索 |

### `_rank_actions_from_tree()` 实现

```python
def _rank_actions_from_tree(
    result: RolloutResult,
    current_id: int,
    strategy: str,
    epsilon: float,
    rng: np.random.RandomState,
    state_trans: Dict[str, Dict],
) -> List[str]:
    current = result.nodes[current_id]
    children = [result.nodes[cid] for cid in current.children_ids]

    # 按 action 分组
    action_groups: Dict[str, List[RolloutNode]] = defaultdict(list)
    for child in children:
        if child.action:
            action_groups[child.action].append(child)

    # 计算每组子树指标
    action_metrics = {}
    for action, child_nodes in action_groups.items():
        descendants = _get_all_descendant_ids(result.nodes, [n.id for n in child_nodes])
        all_ids = descendants + [n.id for n in child_nodes]
        all_nodes = [result.nodes[nid] for nid in all_ids]

        action_metrics[action] = {
            "beam_rank": min(n.beam_id for n in child_nodes if n.beam_id is not None),
            "max_quality": max(n.quality_score for n in all_nodes),
            "avg_win_rate": np.mean([n.win_rate for n in all_nodes]),
            "avg_future_reward": np.mean([n.avg_future_reward for n in all_nodes]),
            "transition_prob": sum(
                sum(state_trans.get(a, {}).get("next_states", {}).values())
                for a in [action]
            ),
        }

    # 按策略排序
    items = list(action_metrics.items())
    if strategy == "best_beam":
        items.sort(key=lambda x: x[1]["beam_rank"])
    elif strategy == "best_subtree_quality":
        items.sort(key=lambda x: x[1]["max_quality"], reverse=True)
    elif strategy == "best_subtree_winrate":
        items.sort(key=lambda x: x[1]["avg_win_rate"], reverse=True)
    elif strategy == "highest_transition_prob":
        items.sort(key=lambda x: x[1]["transition_prob"], reverse=True)
    elif strategy == "random_beam":
        rng.shuffle(items)
    elif strategy == "epsilon_greedy":
        if rng.random() < epsilon and len(items) > 1:
            rng.shuffle(items)
        else:
            items.sort(key=lambda x: x[1]["beam_rank"])

    return [action for action, _ in items]
```

## 8. 关键设计决策记录

| # | 决策点 | 结论 | 理由 |
|---|---|---|---|
| 1 | next_state 与 beam 预测不一致 | 保留 beam 预测子树为探索分支，创建新节点重新束搜索 | 原有分支不再适用，但保留用于追溯分析 |
| 2 | 新增动作选择策略 | 新增 `best_subtree_quality` 和 `best_subtree_winrate` | 利用前瞻路径的多维信息 |
| 3 | 执行顺序 | Phase 1 → Phase 2 → Phase 3 | 先建核心再建 UI |
| 4 | 数据结构 | `RolloutResult` 替代 `List[RolloutStep]`，主路径从树推导 | 统一数据源，避免冗余 |
| 5 | 树节点数量控制 | 50步 × 3beam × 5前瞻 ≈ 750 节点，序列化约 50KB | `@st.cache_data` 可承受 |
