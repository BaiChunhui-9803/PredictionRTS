\begin{abstract}
We present a novel framework for offline model-based decision planning using Experience Transition Graphs (ETG). Unlike conventional reinforcement learning approaches that learn policy functions, our method constructs an explicit graph-structured world model from historical interaction trajectories and performs deterministic beam search planning on this graph. The ETG captures state transition statistics, reward estimates, and quality metrics without requiring neural network training. We introduce a Branch-to-Backup (BTB) recovery mechanism inspired by CPU branch prediction, enabling efficient plan revision when execution diverges from planned trajectories. A four-dimensional path evaluation system combining confidence, quality, win rate, and reward provides balanced action recommendations. Experiments in a real-time strategy game environment demonstrate that our approach achieves competitive performance while maintaining full interpretability of decision paths. The complete decoupling of data generation and planning enables deterministic, reproducible decisions with controllable computational costs.
\end{abstract}

\section{Introduction}

Deep reinforcement learning has achieved remarkable success in complex decision-making domains, from game playing to robotics. However, these approaches often suffer from two critical limitations: the ``black box'' nature of learned policies makes decision reasoning opaque, and the tight coupling between learning and execution complicates system maintenance and debugging.

We propose a fundamentally different approach: instead of learning a policy function, we explicitly construct a world model from experience data and perform planning through graph search. Our framework, called \textbf{Structured Decision Planning Based on Experience Transition Graphs (ETG)}, occupies a unique position in the space of decision-making paradigms—it is offline, model-based, and planning-oriented, distinguishing it from both online reinforcement learning and offline policy learning methods.

The core contribution is the Experience Transition Graph (ETG), a directed weighted graph where nodes represent abstract states and edges encode actions with statistical properties derived from historical interactions. Key attributes include visit frequencies (approximating policy probabilities), average rewards (approximating reward functions), and quality scores (approximating value functions). This tabular representation provides a ``minimal explicit world model'' without neural network parameters.

Building on ETG, we develop:
\begin{itemize}
    \item A beam search planning algorithm with multi-dimensional path scoring
    \item A Branch-to-Backup (BTB) recovery mechanism for handling execution divergence
    \item An interactive visualization system for decision path analysis
\end{itemize}

Our approach offers practical advantages: deterministic and reproducible decisions, full interpretability of action recommendations, and predictable computational costs. Experiments demonstrate competitive performance in a real-time strategy game environment.

\section{Related Work}

\subsection{Model-Based Reinforcement Learning}

Model-based reinforcement learning (MBRL) learns environment dynamics to improve sample efficiency. Classical approaches learn transition models $\hat{T}(s'|s,a)$ and reward models $\hat{R}(s,a)$, then plan using model-predictive control (MPC) or value iteration. Recent work explores neural network world models in latent spaces.

Our ETG differs fundamentally: it is a \textit{tabular} world model constructed directly from experience statistics, not learned through gradient descent. This eliminates training instability and hyperparameter sensitivity while providing exact interpretability \cite{zhu2023vmg}.

\subsection{Offline Reinforcement Learning}

Offline RL learns policies from fixed datasets without environment interaction, addressing challenges of sample efficiency and safety. Representative methods include Conservative Q-Learning (CQL) and behavior-constrained approaches.

Our framework belongs to the less-explored \textit{offline model-based planning} paradigm. Rather than learning a policy function, we plan directly on an empirical transition model. This aligns with Model-Based Offline Planning (MBOP) \cite{argenson2021mbop} but uses explicit graph structures instead of neural dynamics models.

\subsection{Graph-Structured World Models}

Recent work explores graph representations for world models. The Value Memory Graph (VMG) \cite{zhu2023vmg} constructs a directed graph MDP from offline data, while L3P (Learning Latent Landmarks for Planning) \cite{zhang2021l3p} builds sparse graphs from Q-functions.

ETG shares the graph structure but differs in construction and usage: (1) nodes are discretized abstract states via clustering, (2) edges store comprehensive statistics beyond transition probabilities, and (3) planning uses deterministic beam search with multi-objective scoring.

\subsection{State Abstraction}

State abstraction reduces MDP complexity by aggregating similar states \cite{abel2018abstraction}. Methods include bisimulation, model-irrelevant abstraction, and spectral methods \cite{starre2022survey}. Our framework employs BK-Tree hierarchical clustering for continuous state discretization, trading representational precision for interpretability and computational efficiency.

\section{Experience Transition Graph}

\subsection{Formal Definition}

An Experience Transition Graph (ETG) is a directed weighted graph $G = (V, E, \phi)$ where:

\begin{itemize}
    \item $V$ is a finite set of abstract state IDs (discretized from continuous states)
    \item $E \subseteq V \times \mathcal{A} \times V$ is the set of action-labeled directed edges
    \item $\phi: E \rightarrow \mathbb{R}^k$ maps edges to $k$-dimensional statistical attributes
\end{itemize}

\subsection{Data Structure}

The ETG is implemented as a two-level dictionary:

\begin{equation}
\text{ETG} = \begin{cases}
\text{state\_action\_map}: & (s, a) \rightarrow \text{ActionStats} \\
\text{transitions}: & s \rightarrow a \rightarrow s' \rightarrow \text{count}
\end{cases}
\end{equation}

where ActionStats contains:

\begin{align}
\text{visits}(s,a) &= N(s,a) \approx P(a|s) \\
\text{avg\_step\_reward}(s,a) &= \bar{r}(s,a) \approx R(s,a) \\
\text{avg\_future\_reward}(s,a) &= \bar{G}(s,a) \approx Q(s,a) \\
\text{win\_rate}(s,a) &= \text{success ratio} \\
\text{quality\_score}(s,a) &= f(\bar{r}, \bar{G}, \text{win\_rate})
\end{align}

\subsection{Probabilistic Semantics}

ETG attributes provide empirical approximations to MDP quantities:

\begin{table}[h]
\centering
\begin{tabular}{lll}
\toprule
\textbf{Attribute} & \textbf{Empirical Estimate} & \textbf{MDP Semantics} \\
\midrule
visits$(s,a)$ & $N(s,a)$ & $\pi(a|s)$ \\
transitions$(s,a,s')$ & $N(s,a,s')/N(s,a)$ & $T(s'|s,a)$ \\
avg\_step\_reward$(s,a)$ & $\bar{r}(s,a)$ & $R(s,a)$ \\
avg\_future\_reward$(s,a)$ & $\bar{G}(s,a)$ & $Q(s,a)$ \\
\bottomrule
\end{tabular}

\end{table}

\subsection{Construction Pipeline}

ETG construction follows three stages:

\textbf{Stage 1: Trajectory Collection.} An RL agent interacts with the environment, recording transition tuples $(s, a, r, s')$. States are continuous observations from the environment.

\textbf{Stage 2: State Abstraction.} Continuous states are discretized using BK-Tree hierarchical clustering, mapping each state to an abstract state ID.

\textbf{Stage 3: Statistical Aggregation.} Transition statistics are aggregated:
\begin{algorithmic}[1]
\FOR{each transition $(s, a, r, s')$}
    \STATE $s_{id} \leftarrow \text{Discretize}(s)$
    \STATE $s'_{id} \leftarrow \text{Discretize}(s')$
    \STATE Increment visits$(s_{id}, a)$
    \STATE Increment transitions$(s_{id}, a, s'_{id})$
    \STATE Update running statistics for rewards
\ENDFOR
\end{algorithmic}

\section{Beam Search Planning}

\subsection{Algorithm}

Given a starting state $s_0$, beam search explores the ETG to find high-quality action sequences:

\begin{algorithm}[t]
\caption{Beam Search on ETG}
\label{alg:beam}
\textbf{Input}: $s_0$, beam width $K$, max depth $D$, scoring function $f$ \\
\textbf{Output}: recommended action $a^*$
\begin{algorithmic}[1]
\STATE Initialize beam $B_0 \leftarrow \{(s_0, \text{score}=0, \text{path}=[])\}$
\FOR{$d = 1$ to $D$}
    \STATE $B_d \leftarrow \emptyset$
    \FOR{each $(s, c, p) \in B_{d-1}$}
        \FOR{each action $a$ available at $s$}
            \STATE $\phi \leftarrow \text{GetActionStats}(s, a)$
            \STATE $\text{score}_{\text{new}} \leftarrow f(c, \phi) \times \gamma^d$
            \FOR{each reachable $s'$ with $P(s'|s,a) > \theta$}
                \STATE Add $(s', \text{score}_{\text{new}}, p + [(s, a)])$ to $B_d$
            \ENDFOR
        \ENDFOR
    \ENDFOR
    \STATE Prune: retain top-$K$ paths by cumulative score
    \IF{all paths reach terminal states}
        \STATE \textbf{break}
    \ENDIF
\ENDFOR
\STATE \textbf{return} action $a^*$ from highest-scored path
\end{algorithmic}
\end{algorithm}

\subsection{Scoring Strategies}

We support three scoring strategies for different planning objectives:

\begin{itemize}
    \item \textbf{Quality Score}: $f = \text{quality\_score}(s,a)$ — balanced multi-objective optimization
    \item \textbf{Future Reward}: $f = \text{avg\_future\_reward}(s,a)$ — long-term return maximization
    \item \textbf{Win Rate}: $f = \text{win\_rate}(s,a)$ — success probability maximization
\end{itemize}

\subsection{Four-Dimensional Path Evaluation}

Each candidate path is evaluated along four dimensions:

\begin{enumerate}
    \item \textbf{Confidence}: $\prod_i P(s_{i+1}|s_i, a_i)$ — empirical reliability
    \item \textbf{Quality}: average quality\_score along path
    \item \textbf{Win Rate}: terminal state success probability
    \item \textbf{Reward}: cumulative immediate rewards
\end{enumerate}

The composite score combines normalized values:
\begin{equation}
\text{Score} = \sum_{i=1}^{4} w_i \cdot \text{Normalize}(D_i)
\end{equation}
where weights $w_i$ can be adjusted for task requirements.

\section{Receding-Horizon Planning with BTB Recovery}

\subsection{Plan-Execute-Diverge Loop}

Single-step planning extends to continuous execution through a receding-horizon framework:

\begin{algorithm}[t]
\caption{Receding-Horizon Rollout}
\label{alg:rollout}
\textbf{Input}: $s_0$, plan depth $D$, beam width $K$, switching map $M$ \\
\textbf{Output}: trajectory $\tau$
\begin{algorithmic}[1]
\STATE $s \leftarrow s_0$, $\tau \leftarrow []$
\WHILE{not terminal}
    \STATE \textbf{PLAN}: $\text{plan} \leftarrow \text{BeamSearch}(s, D)$
    \STATE $\text{planned\_path} \leftarrow \text{plan.best\_path}$
    \FOR{each $(s_i, a_i)$ in planned\_path}
        \STATE Execute $a_i$, observe $s'$
        \STATE Append $(s_i, a_i, s')$ to $\tau$
        \IF{$s' \approx$ planned next state}
            \STATE Continue along plan
        \ELSE
            \STATE \textbf{DIVERGE}: $s \leftarrow \text{BTB\_Recovery}(s', M)$
            \STATE \textbf{break}
        \ENDIF
    \ENDFOR
\ENDWHILE
\end{algorithmic}
\end{algorithm}

\subsection{Branch-to-Backup Recovery}

Inspired by CPU branch prediction mechanisms, we precompute backup switching points:

\textbf{Precomputation Phase}:
\begin{enumerate}
    \item For each state $s$, evaluate all candidate actions
    \item Identify switching points where alternative high-quality paths diverge
    \item Store switching map: $M: s \rightarrow \text{switching\_points}$
\end{enumerate}

\textbf{Recovery Phase}:
\begin{enumerate}
    \item Exact match: find switching point with identical state ID
    \item Distance fallback: find nearest switching point within threshold
    \item Switch to backup path without full replanning
\end{enumerate}

This mechanism reduces computational overhead during execution while maintaining robustness to deviations.

\section{Experiments}

\subsection{Experimental Setup}

We evaluate our framework in a real-time strategy (RTS) game environment with the following characteristics:

\begin{itemize}
    \item State space: continuous 256-dimensional observations
    \item Action space: discrete, 18 actions
    \item Reward: sparse win/lose signals + dense step rewards
    \item Episode length: up to 500 steps
\end{itemize}

\textbf{Data Collection}: A trained RL agent generates 10,000 episodes, from which we construct the ETG.

\textbf{Baselines}:
\begin{itemize}
    \item PPO agent (online policy) \cite{schulman2017ppo}
    \item Behavior cloning from same dataset \cite{fujimoto2021bc}
    \item CQL (offline RL) \cite{kumar2020cql}
    \item Model-based offline planning (MBOP) \cite{argenson2021mbop}
\end{itemize}

\section{Discussion}

\subsection{Advantages}

\textbf{Interpretability}: Every decision is traceable through the ETG structure. Users can inspect why an action was recommended and what alternatives were considered.

\textbf{Determinism}: Planning on a fixed ETG produces identical results for identical inputs, enabling reproducible debugging and analysis.

\textbf{Decoupled Architecture}: Data generation (RL interaction) and decision making (graph search) are completely separate, allowing independent optimization of each component.

\textbf{Computational Predictability}: Planning time is bounded by beam width and search depth, without the variability of neural network inference.

\subsection{Limitations}

\textbf{State Discretization}: The discretization process loses fine-grained state distinctions, potentially missing nuanced decision boundaries.

\textbf{Coverage Dependency}: ETG quality depends on exploration coverage during data collection. Novel state-action pairs have no statistics.

\textbf{Scalability}: Tabular representation may not scale to extremely large state spaces without further abstraction.

\subsection{Future Directions}

\begin{itemize}
    \item Hierarchical ETG for multi-scale planning
    \item Online ETG updates for adaptive planning
    \item Integration with neural world models for hybrid approaches
\end{itemize}

\section{Conclusion}

We presented Structured Decision Planning based on Experience Transition Graphs, a framework for interpretable offline decision making. By constructing explicit graph-structured world models from interaction data and performing deterministic beam search planning, our approach achieves competitive performance while maintaining full transparency of decision processes. The BTB recovery mechanism and four-dimensional path evaluation provide robust and balanced action recommendations. This work demonstrates that explicit model-based planning remains a viable and valuable paradigm in the era of deep reinforcement learning.
