# 本项目技术路线的学术术语严谨性分析

> 分析报告 | 2026-04-10

---

## 1. 问题缘起

本项目三份技术文档中使用了"决策知识图谱（DKG）"、"经验知识图谱"等术语来描述系统的核心数据结构。本报告从学术溯源出发，逐层剖析该技术路线的实质，并给出严谨准确的术语建议。

---

## 2. 当前术语的问题诊断

### 2.1 "决策知识图谱"的学术溯源

"Decision Knowledge Graph"（DKG）并非学术界的通用术语。经检索，该术语仅在一篇论文中被作为专有名词提出：

- **Kandula & Bhattacharyya (2023)**，*"Decision Knowledge Graphs: Construction of and Usage in Question Answering for Clinical Practice Guidelines"*（arXiv: 2308.02984）
- 其定义：在传统知识图谱上增加"决策维度"（条件-分支-行动结构），专门用于表示临床实践指南中的决策流程
- 作者自述为首次尝试

**关键问题**：该定义（在 KG 中显式编码条件-分支决策逻辑）与本项目的技术路线**并不匹配**。本系统并不在图谱中嵌入 if-else 决策规则，而是在转移图上执行 Beam Search 搜索——两者的技术本质完全不同。

### 2.2 "经验知识图谱"的学术溯源

"Experience Knowledge Graph" 在学术数据库中**无任何出处**，属于完全自造的概念。

### 2.3 调研文档中的错误引用

第三份调研文件（`调研-知识图谱与强化学习.md`）第49行声称：

> "常被称为'决策知识图谱'（Decision Knowledge Graph）或'经验图谱'"

这一表述不准确。这两个术语均非该领域的通用或权威命名，将其包装为"常见说法"存在学术风险。

---

## 3. 技术本质的逐层学术定位

### 3.1 本项目做了什么？

从三份技术文档中提取系统核心流水线：

```
RL 训练 → 轨迹采样 (s,a,r,s') → 状态离散化 → 统计 state→action 的经验指标
→ 构建有向带权图 → Beam Search 多步搜索 → Plan-Execute-Diverge 推演 → 可视化
```

逐层拆解每一环节的学术实质：

| 层次 | 本项目的实现 | 学术本质 | 准确术语 |
|------|-------------|----------|----------|
| 数据采集 | RL 智能体与环境交互，记录 $(s, a, r, s')$ 序列 | MDP 环境下的轨迹采样 | MDP Transition Trajectories |
| 状态表示 | 连续状态经离散化映射为有限状态集 | 状态抽象/离散化 | State Abstraction / State Discretization |
| 图结构构建 | 统计 $(s, a)$ 的 visits、win_rate、quality_score 等指标，记录 $s \xrightarrow{a} s'$ 的转移频次 | 从经验数据构建转移模型 | Experience-Based Transition Model |
| 图上查询 | 支持基于历史动作窗口 $(s, h)$ 的上下文感知查询 | 高阶 Markov 近似 | High-Order Markov Lookup |
| 推理决策 | Beam Search 在转移图上做多步受限搜索 | 确定性规划/启发式图搜索 | Heuristic Planning on Graphs / Deterministic Planning |
| 多步推演 | Plan-Execute-Diverge 循环 + 备选路径切换（BTB） | 带偏差恢复的滚动规划 | Receding-Horizon Planning with Recovery |
| 整体范式 | 数据生产（RL）与决策推理（图搜索）完全解耦 | 离线模型规划 | Offline Model-Based Planning |

### 3.2 核心数据结构的学术定位

系统的核心数据结构是 `DecisionKnowledgeGraph`，其实际内容为：

```
KG = { (s, a) → ActionStats }  // 两级字典
ActionStats = { visits, avg_step_reward, avg_future_reward, win_rate, quality_score }
转移图 = { s → { a → { s' → count } } }  // 状态转移计数
```

**学术定位**：这是一个**带经验统计属性的经验转移图（Experience Transition Graph）**。

具体特征：
- **节点**：抽象状态 ID（非语义实体）
- **边**：动作 + 转移频次（非语义关系谓词）
- **属性**：频率统计 → 近似概率，累积奖励 → 近似效用（非事实真值）

它与以下经典概念的关系：

| 概念 | 与本项目的关系 |
|------|--------------|
| **Markov Decision Process (MDP)** | 本系统的转移图是对 MDP 转移函数 $P(s'|s,a)$ 和奖励函数 $R(s,a)$ 的**经验频率近似** |
| **Transition Graph** | 本系统的图结构就是标准的有向带权转移图 |
| **Experience Replay / Replay Buffer** | 本系统从 RL 轨迹中提取经验，但目标不是训练神经网络，而是构建转移统计——这更接近**表格型 RL 的经验聚合** |
| **World Model** | 本系统的转移图是一种最简形式的 World Model：显式存储状态转移而不学习神经网络 |

### 3.3 整体范式的学术归属

第二份报告第1.2节已经给出了非常好的定位分析，此处进一步细化：

**最准确的学术领域定义：Offline Model-Based Planning on Experience Transition Graphs**

理由：
1. **Offline**：模型（转移图）在推理阶段固定不变，不从环境获取新数据
2. **Model-Based**：系统显式建模了环境的转移函数（转移图）和奖励函数（质量分），而非端到端学习策略
3. **Planning**：决策过程是"给定当前状态，在模型中搜索最优动作序列"，即经典的规划问题
4. **Experience Transition Graphs**：模型来源于历史经验的统计聚合，而非解析模型或学习模型

与相关领域的对比定位：

```
                    端到端策略学习          模型规划
                  (Policy Learning)    (Model-Based Planning)
                  ─────────────────    ──────────────────────
在线交互           Online RL            ── (不常见)
                  (DQN, PPO, ...)      

离线固定模型       ── (不常见)          ★ 本系统所在位置
                                       Offline Model-Based Planning
                                       (转移图 + Beam Search)

离线数据集         Offline RL          ── (不常见)
                  (CQL, TD3+BC, ...)
```

本系统的独特性在于：它占据了一个**不太常见但完全合理**的学术位置——离线、显式模型、基于搜索的规划。这不同于 Offline RL（从离线数据集学习策略函数），也不同于 Online RL（边交互边学习）。

### 3.4 与知识图谱的本质区别

知识图谱（Knowledge Graph）的核心语义是**实体-关系三元组**（Subject-Predicate-Object），其"知识"指的是**语义事实**（如"爱因斯坦出生于乌尔姆"），其标准能力包括：

- 实体消解与链接
- 关系推理与链接预测
- 本体论与层次化分类
- 语义查询（SPARQL 等）

本系统的"图谱"不具备上述任何一项能力。三份文档中试图论证"状态即实体、动作即关系"的映射关系，但这是一种**隐喻性的类比**，而非学术上的等价：

| 知识图谱（KG） | 本系统的转移图 |
|--------------|--------------|
| 节点是语义实体（人、地点、概念） | 节点是抽象状态 ID（离散化编码） |
| 边是语义关系（is-a, located-in） | 边是动作（物理操作指令） |
| 属性是事实真值（布尔或数值事实） | 属性是统计量（频率、均值、概率） |
| 查询是语义推理（多跳推理、链接预测） | 查询是图搜索（Beam Search、路径回溯） |
| 知识来源是文本/结构化数据/本体论 | 知识来源是 RL 交互轨迹 |

**结论**：将本系统的核心数据结构称为"知识图谱"是一种**不恰当的概念借用**。它本质上就是一个带权有向转移图，附带经验统计属性。硬套"知识图谱"名称不仅不会提升学术高度，反而会因为术语不准确而引发审稿人的质疑。

## 4. 推荐术语体系

### 4.1 首选术语

> **基于经验转移图的结构化决策规划框架**
> *(Structured Decision Planning Framework based on Experience Transition Graphs)*

### 4.2 分场景替代表述

| 场景 | 推荐术语 | 说明 |
|------|----------|------|
| 描述核心数据结构 | **经验转移图** (Experience Transition Graph) | 准确反映"从经验构建的转移图"这一本质 |
| 描述学术领域归属 | **离线模型规划** (Offline Model-Based Planning) | 对应 planning community 的标准分类 |
| 描述搜索算法 | **图约束束搜索规划** (Beam Search Planning on Transition Graphs) | 强调算法层面的创新 |
| 描述与 RL 的关系 | **从 RL 轨迹提取的决策模型** (Decision Model Extracted from RL Trajectories) | 用于动机/背景叙述 |
| 需要更宽泛的表述 | **基于经验图谱的决策推理** | "图谱"比"知识图谱"宽泛，不预设语义能力 |
| 描述整体系统 | **交互式经验转移图决策推演系统** | 保留"推演"一词体现 Plan-Execute-Diverge 特性 |

### 4.3 不建议继续使用的术语

| 术语 | 不建议的原因 |
|------|-------------|
| 决策知识图谱 (DKG) | 非标准术语，仅1篇论文使用且含义不匹配；Kandula (2023) 的 DKG 指在 KG 中嵌入条件分支逻辑，与本项目不同 |
| 经验知识图谱 | 完全无学术出处，自造概念 |
| 知识图谱辅助决策 | 低估了系统的核心作用（图谱本身就是决策模型，而非"辅助"角色） |

## 6. 本项目在学术谱系中的优势位置

放弃"知识图谱"这个不准确的标签后，本项目的学术定位反而更加清晰和有优势：

**本项目的核心贡献是：在 Offline Model-Based Planning 这一范式下，针对 RTS 游戏的高维连续状态空间，提出了一套完整的状态抽象 → 经验转移图构建 → 图上 Beam Search 规划 → Plan-Execute-Diverge 偏差恢复的技术方案，并实现了交互式可视化分析。**

这一贡献的真实学术价值在于：
1. **可解释性**：决策路径在转移图上完全显式可追溯，解决了深度 RL 的黑盒问题
2. **推理与学习的解耦**：不依赖策略函数的在线迭代，规划过程是确定性的
3. **偏差恢复机制（BTB）**：借鉴 CPU 分支预测思想的备选路径切换，是规划领域的创新
4. **四维路径评估**：置信度 + 质量 + 胜率 + 奖励的综合评分体系

这些贡献完全可以在 **Model-Based Planning / Planning in Games / Game AI** 等标准学术语境下被准确理解和评价，无需借助"知识图谱"的标签。
