> *Structured Decision Planning Framework based on Experience Transition Graphs*
>
> 版本 | 2026-04-10

## 1. 学术定位

本系统的最准确学术领域定义为：

> **Offline Model-Based Planning on Experience Transition Graphs**

| 维度 | 属性 | 说明 |
|------|------|------|
| **Offline** | 模型在推理阶段固定不变，不从环境获取新数据 | 区别于 Online RL 的边交互边学习 |
| **Model-Based** | 显式建模环境的转移函数（转移图）和奖励函数（质量分），而非端到端学习策略 | 区别于 Model-Free RL |
| **Planning** | 决策过程是"给定当前状态，在模型中搜索最优动作序列"，即经典的规划问题 | 区别于 Offline RL（学习策略函数） |
| **Experience Transition Graphs** | 模型来源于历史 RL 交互轨迹的统计聚合，而非解析模型或学习模型 | 最简形式的显式 World Model |

### 与相关领域的对比定位

```
                    端到端策略学习          模型规划
                  (Policy Learning)    (Model-Based Planning)
                  ─────────────────    ──────────────────────
在线交互           Online RL            ── (不常见)
                  (DQN, PPO, ...)

离线固定模型       ── (不常见)          ★ 本系统所在位置
                                        Offline Model-Based Planning
                                        (转移图 + Beam Search)

离线数据集         Offline RL           ── (不常见)
                  (CQL, TD3+BC, ...)
```

本系统的独特性在于：它占据了一个**不太常见但完全合理**的学术位置——离线、显式模型、基于搜索的规划。这不同于 Offline RL（从离线数据集学习策略函数），也不同于 Online RL（边交互边学习）。

## 3. 核心数据结构：经验转移图

### 3.1 数学定义

经验转移图 (Experience Transition Graph, ETG) 是一个带权有向图 $G = (V, E, \phi)$：

- **节点集** $V$：抽象状态 ID 集合（离散化编码，非语义实体）
- **边集** $E \subseteq V \times A \times V$：动作标签的有向边（$A$ 为动作空间）
- **属性函数** $\phi: E \to \mathbb{R}^k$：边上附带的 $k$ 维统计属性

### 3.2 数据结构实现

```
ETG = {
    state_action_map: Dict[state_id, Dict[action, ActionStats]],
    transitions:      Dict[state_id, Dict[action, Dict[next_state_id, int]]]
}

ActionStats = {
    visits:            int,     // (s, a) 访问次数 → 近似 P(a|s)
    avg_step_reward:   float,   // 即时奖励均值 → 近似 R(s, a)
    avg_future_reward: float,   // 未来累积奖励均值 → 近似 V(s, a)
    win_rate:          float,   // 胜率 → 近似策略质量
    quality_score:     float,   // 综合质量评分 → 近似效用函数
}

transitions:  s → a → s' → count    // 状态转移计数 → 近似 P(s'|s, a)
```

### 3.3 统计属性的概率语义

| 属性 | 统计量 | 概率语义 | 近似目标 |
|------|--------|----------|----------|
| `visits(s, a)` | 访问频次 $N(s, a)$ | $P(a \mid s) \propto N(s, a)$ | MDP 策略 $\pi(a \mid s)$ |
| `transitions(s, a, s')` | 转移频次 $N(s, a, s')$ | $P(s' \mid s, a) = N(s, a, s') / N(s, a)$ | MDP 转移函数 $T(s' \mid s, a)$ |
| `avg_step_reward(s, a)` | 即时奖励均值 $\bar{r}(s, a)$ | $R(s, a) \approx \bar{r}(s, a)$ | MDP 奖励函数 $R(s, a)$ |
| `avg_future_reward(s, a)` | 未来奖励均值 $\bar{G}(s, a)$ | $Q(s, a) \approx \bar{G}(s, a)$ | MDP 状态-动作值 $Q^*(s, a)$ |
| `quality_score(s, a)` | 综合评分 | $\text{quality}(s, a) = f(\bar{r}, \bar{G}, \text{win\_rate})$ | 效用函数 $U(s, a)$ |

### 3.4 与经典概念的关系

| 概念 | 与 ETG 的关系 |
|------|--------------|
| **MDP 转移模型** | ETG 是对 MDP 转移函数 $P(s'\|s,a)$ 和奖励函数 $R(s,a)$ 的经验频率近似 |
| **Transition Graph** | ETG 在结构上就是标准的有向带权转移图 |
| **World Model** | ETG 是最简形式的显式 World Model：直接存储状态转移，不学习神经网络参数 |
| **表格型 RL 经验聚合** | ETG 从 RL 轨迹中提取经验并聚合为查找表，但目标不是在线学习，而是离线规划 |

## 4. 规划算法

### 4.1 束搜索规划 (Beam Search Planning)

给定起始状态 $s_0$，在 ETG 上执行受限的多步前向搜索：

```
Algorithm: Beam Search on ETG
Input: s₀, beam_width K, max_depth D, scoring_function f
Output: recommended_action a*

1.  Initialize beam B₀ = {(s₀, score=0, path=[])}
2.  For depth d = 1, 2, ..., D:
3.      Expand each node (s, c, p) ∈ B_{d-1}:
4.          For each action a available at s:
5.              Get ActionStats φ(s, a), transition probabilities P(s'|s, a)
6.              score_new = f(c, φ) × discount^d
7.              For each reachable s' with prob > threshold:
8.                  Add (s', score_new, p + [(s, a)])
9.      Prune: retain top-K paths by cumulative score → B_d
10.     If all paths reach terminal states: break
11. Return action a* from highest-scored path in B_D
```

#### 评分策略

系统支持三种评分策略，对应不同的规划目标：

| 策略 | 公式 | 适用场景 |
|------|------|----------|
| **Quality Score** | $f = \text{quality\_score}(s, a)$ | 综合质量优先 |
| **Future Reward** | $f = \text{avg\_future\_reward}(s, a)$ | 长期收益优先 |
| **Win Rate** | $f = \text{win\_rate}(s, a)$ | 胜率优先 |

#### 路径评估维度

每条候选路径从四个维度评估：

1. **置信度 (Confidence)**：路径累积转移概率 $\prod_{i} P(s_{i+1} \mid s_i, a_i)$，反映路径的经验可靠性
2. **质量 (Quality)**：路径节点平均 quality_score，反映动作选择质量
3. **胜率 (Win Rate)**：路径终点的胜率统计，反映最终收益
4. **奖励 (Reward)**：路径累积即时奖励，反映每步收益

综合评分通过 Min-Max 归一化后的加权和计算。

### 4.2 滚动推演 (Receding-Horizon Planning with Recovery)

滚动推演将束搜索从单次规划扩展为持续的 Plan-Execute-Diverge 循环：

#### 单步模式 (Single-Step Rollout)

```
Algorithm: Single-Step Rollout
Input: s₀, beam_width K, scoring f, action_strategy, state_transition_mode
Output: trajectory [(s₀, a₀, r₀), (s₁, a₁, r₁), ..., (s_T, ⊥, r_T)]

1.  current_state ← s₀
2.  For each step t = 0, 1, ..., max_steps:
3.      Run Beam Search from current_state → get recommended action a*
4.      Execute a* via transition model (sample or highest-prob)
5.      Record (current_state, a*, reward)
6.      current_state ← next_state
7.      If terminal: break
```

#### 多步模式 (Multi-Step Rollout)

多步模式引入**计划-执行-偏转 (Plan-Execute-Diverge)** 机制：

```
Algorithm: Multi-Step Rollout with BTB Recovery
Input: s₀, plan_depth, beam_width, switching_map, score_threshold, distance_threshold
Output: trajectory with switching points

1.  current_state ← s₀
2.  While not terminal:
3.      // PLAN: 在 ETG 上执行 plan_depth 步束搜索
4.      plan ← Beam Search(current_state, depth=plan_depth)
5.      planned_path ← plan.best_path
6.
7.      // EXECUTE: 沿规划路径逐步执行
8.      For each (s, a) in planned_path:
9.          Execute a, observe actual next_state s'
10.         If s' ≈ planned next_state (within threshold):
11.             Continue along plan
12.         Else:
13.             // DIVERGE: 实际状态偏离规划
14.             If backup_switch enabled:
15.                 Find closest switch point in switching_map
16.                 Switch to backup path via BTB (Branch-to-Backup)
17.                 Re-plan from diverge state
18.             Else:
19.                 Re-plan from actual state immediately
20.             Break execution loop
```

#### BTB 偏差恢复 (Branch-to-Backup)

借鉴 CPU 分支预测中的"分支目标缓冲"思想：

- **预计算**：在推演前，对 ETG 中每个状态节点预计算备选路径切换点（switching_map）
- **切换点构建**：对每个状态，评估所有可选动作的路径质量，筛选评分超过阈值的备选路径，并按分叉点去重
- **偏差匹配**：采用精确优先 + 距离回退的两阶段策略，在实际状态偏离规划时快速定位最近的切换点
- **无缝切换**：命中切换点后直接跳转到备选路径，避免完全从头重新规划的开销

## 6. 核心创新点

### 6.1 可解释的决策路径

决策路径在 ETG 上完全显式可追溯。每一步的状态转移、动作选择、评分依据均可通过路径树图和详情表直接查看。这从根本上解决了深度 RL 的黑盒问题——用户不仅知道"推荐什么动作"，还知道"为什么推荐这个动作"以及"如果换一个动作会怎样"。

### 6.2 数据生产与决策规划的解耦

本系统实现了 RL 数据采集与决策规划的完全解耦：

- **数据阶段**：RL 智能体在环境中自由探索，生成 $(s, a, r, s')$ 轨迹
- **建模阶段**：将轨迹聚合为经验转移图，无需神经网络训练
- **规划阶段**：在固定的 ETG 上执行确定性搜索，不依赖策略函数的在线迭代

这种解耦意味着：(1) 规划过程是确定性的，同一状态总是返回相同结果；(2) 模型更新与规划算法可以独立演进；(3) 规划计算量可控且可预测。

### 6.3 BTB 偏差恢复机制 (Branch-to-Backup Recovery)

借鉴 CPU 分支预测中"分支目标缓冲"的设计思想，提出了一种新颖的规划偏差恢复策略：

- **预计算备选路径**：在推演前构建 switching_map，为每个状态预计算高质量的备选路径切换点
- **两阶段偏差匹配**：精确优先（状态 ID 完全匹配）+ 距离回退（连续空间距离阈值），兼顾精确性和鲁棒性
- **无缝切换**：命中切换点后直接跳转到备选路径，避免完全从头重新规划的计算开销
- **动态裁剪**：通过评分阈值和分叉点去重控制备选路径的规模和质量

### 6.4 四维路径评估体系

每条候选路径从置信度、质量、胜率、奖励四个维度独立评估，通过 Min-Max 归一化后的加权和计算综合评分。这种多维评估避免了单一指标（如纯概率或纯奖励）的片面性，使推荐结果更加均衡和可靠。
