# PredictionRTS 项目实验架构总结

## 一、动机分析

### 1.1 项目功能

项目围绕 **即时战略游戏（RTS）中的智能体行为预测与决策**，主要做三件事：

1. **数据采集与结构化**  
   在 StarCraft II（PySC2）中运行分层 Q-Learning 智能体（`SmartAgent`），采集对战轨迹：状态序列（state_log）、动作序列（action_log）、奖励序列（r_log）、游戏结果（game_result）等，并写入 `datas/data_for_transit` 或外部指定路径。

2. **轨迹/状态建模与预测**  
   - **状态序列预测**：用 Transformer 类模型（TrajectoryTransformer / SGTransformer）对“状态索引序列”做自回归预测（给定前 N 步预测后 K 步）。  
   - **决策预测**：用 Decision Transformer（DT）对“状态 + 动作 + 回报”三元组建模，预测下一步动作。

3. **分析与可视化**  
   基于 BK-Tree 与自定义状态距离，计算状态距离矩阵、轨迹 DTW 距离、适应度景观（Fitness Landscape）、MDS 降维与插值地形等，并导出 Excel/PDF 图表和 LaTeX 技术方案。

### 1.2 研究动机

- **问题**：RTS 状态/轨迹高维、离散、带时序与空间结构，直接端到端学习困难。  
- **思路**：  
  - 用 **BK-Tree + 聚类** 将原始状态离散化为“状态节点/聚类 ID”，得到可建模的离散序列；  
  - 用 **自定义状态距离**（坐标+生命值、匈牙利匹配等）和 **DTW** 刻画状态/轨迹相似性；  
  - 用 **Transformer** 做序列/决策建模，并引入 **距离诱导的表征**（MDS 初始化、SG 注意力偏置、空间先验等），使几何/距离信息进入模型。  
- **目标**：在离线轨迹上学习“状态演化”与“状态→动作”的映射，为 RTS 中的轨迹预测、策略分析与决策生成提供可解释、可评估的模型。

## 二、核心 Pipeline

### 2.1 从原始数据到结果的整体流程

```
Raw Data (PySC2 对局)
    ↓
[数据采集] main.py / env/starcraft.py
    SmartAgent + Q-Learning → 写入 datas/data_for_transit
    (game_result.txt, node_log.txt, state_node.txt, action_log.csv, sub_episode/*.csv, sub_q_table/*.csv 等)
    ↓
[全局变量与路径] data/global_variable.py
    从 config/base_config 的 map_id, data_id 等派生数据路径（BKTree、graph、distance 等）
    ↓
[离线加载与预处理] data/load_data.py（导入时执行）
    ├─ BK-Tree: load_bk_tree_from_file, load_all_bktrees
    ├─ state_node / node_log: read_state_node_file, read_node_log_file → state_log (轨迹索引序列)
    ├─ 状态距离矩阵: calculate_and_save_distance_matrix (reverse_dict, custom_distance, secondary_bk_trees)
    ├─ 轨迹距离矩阵: calculate_and_save_dtw_distance_matrix (state_log, state_distance_matrix)
    ├─ game_result: read_game_result_file
    ├─ Fitness Landscape: generate_fitness_landscape(game_results, state_log_distance_matrix)
    ├─ State Landscape: get_state_landscape(state_distance_matrix, state_value)
    ├─ action_dict / action_log: create_action_dictionary, read_action_csv
    ├─ episode_results, r_log: batch_process_episodes, process_rewards
    └─ DT 数据: preprocess_decision_transformer_data(state_log, action_log, r_log) → dt_data, action_vocab
    ↓
[模型输入]
    • 状态序列模型: state_log → RLTrajectoryDataset → (input_seq, target_seq)，pad_id，collate_fn
    • DT 模型: dt_data → DTDataset → (states, actions, rtgs, timesteps)，context_window 截断
    ↓
[训练] train/train_state_seq_prediction.py 或 train/train_decision_transformer.py
    • 状态序列: CrossEntropyLoss(ignore_index=PAD_ID)，AdamW，可选 MDS 初始化 / 空间先验 / SG bias
    • DT: CrossEntropyLoss 对 action logits，AdamW，可选 MDS 初始化 state embedding
    ↓
[评估与输出]
    • 状态序列: evaluate_test_performance (按步距离误差)，print_final_comparison → Excel
    • DT: evaluate_action_accuracy_detailed → 准确率、Fitness 关联
    • 图表: plot_prediction, plot_dt, plot_model, plot_scores → output/figures, output/embedding, output/data/
```

### 2.2 两条主入口

- **main.py**：PySC2 环境 + SmartAgent 采集数据，写入 `datas/data_for_transit`（或按配置拷贝到 `datas/data_save/<i>`）。  
- **prediction_RTS.py**：在已加载的全局数据（load_data 导入时已算好）上做 **状态序列训练/预测** 或 **DT 训练/评估/画图**，并可选调用 `env.starcraft.main_run()` 再跑环境。

## 三、数学/算法模块

### 3.1 损失函数

| 位置 | 作用 | 说明 |
|------|------|------|
| `train_state_seq_prediction.train_one_epoch` | 状态序列 | `nn.CrossEntropyLoss(ignore_index=PAD_ID)`，对 logits 与 target 序列做交叉熵 |
| `train_decision_transformer.train_dt` | 决策 | `nn.CrossEntropyLoss`，对 action logits 与真实 action ID |

### 3.2 模型结构

| 模块 | 文件 | 说明 |
|------|------|------|
| **TrajectoryTransformer** | `models/Transformer.py` | Embedding + 可学习位置编码 + `TransformerEncoder`，因果 mask，输出头 `Linear(d_model, vocab_size)`，做下一 token 预测 |
| **SimilarityGuidedTransformer (SGTransformer)** | `models/SGTransformer.py` | 自定义 `GuidedMultiHeadAttention`：在因果 mask 上叠加外部 `sim_bias`（如状态距离诱导的偏置），可学习 `bias_scale`；其余为 LayerNorm + FFN，输出头同 TrajectoryTransformer |
| **DecisionTransformer** | `models/DecisionTransformer.py` | 对 RTG/State/Action 分别嵌入 + 时间步嵌入，按 [R1,S1,A1, R2,S2,A2,...] 拼接，过 `TransformerEncoder`，取 state 位置（1,4,7,...）的输出预测 action logits |

### 3.3 优化器

- **状态序列**：`torch.optim.AdamW(..., lr=lr, weight_decay=0.01)`，梯度裁剪 `clip_grad_norm_(model.parameters(), 1.0)`。  
- **DT**：`torch.optim.AdamW(filter(requires_grad), lr=lr)`。

### 3.4 核心算法与数据结构

| 功能 | 位置 | 说明 |
|------|------|------|
| **BK-Tree** | `structure/ClusterCentricBKTree/BKTree.py` | 按聚类/状态组织树，`custom_distance` 用于节点间距离；`BKTree_sc2.py` 为 SC2 特化 |
| **状态距离** | `structure/state_distance/custom_distance*.py` | 多维距离（坐标+生命值、匈牙利匹配等），`multi_distance` 返回可转为欧氏距离的分量 |
| **状态距离矩阵** | `utils/calculate_utils.calculate_distance_matrix` | 基于 `reverse_dict` 与 secondary BK-Tree 取状态，用 `custom_distance` 两两计算，得到 state_distance_matrix |
| **DTW** | `utils/calculate_utils.dtw_distance` | 基于 state_distance_matrix 的 DTW，用于轨迹间距离；`calculate_dtw_distance_matrix` 得到 log_distance_matrix |
| **Fitness Landscape** | `structure/fitness_landscpae/generate_FL.py` | 用 MDS 将 log_distance_matrix 降维到 2D，再对 game 结果（如 score+penalty）做 griddata 插值得到地形 |
| **State Landscape** | 同上 `get_state_landscape` | 对 state_distance_matrix 做 MDS，对 state_value 插值得到状态价值地形 |
| **MDS 初始化** | `utils/model_utils.init_embedding_with_dm` / `init_dt_state_embedding` | 用状态距离矩阵做 MDS 降维到 d_model，归一化后写入 Transformer 的 embedding 或 DT 的 embed_state |
| **空间先验** | `train_state_seq_prediction.predict_k_steps` | 预测时用 `state_dm` 构造高斯先验，与 logits 相乘再 argmax，使近邻状态更易被选中 |

### 3.5 其他

- **pattern_analysis**（`algorithm/pattern_mining/pattern_analysis.py`）：从 action_log 中做连续子序列挖掘、支持度与模式替换，用于动作序列分析，未在主线训练中直接调用。  
- **Q-Learning / 分层 Q**：`main.py` 中 `QLearningTable`、SmartAgent 的 cluster 与 combat 两层 Q 表，用于环境内策略采集，不属于“预测模型”的数学模块。

## 四、参数管理

### 4.1 方式

- **集中配置**：`config/base_config.py`，Python 变量，无 argparse/Hydra/YAML。
- **派生与路径**：`data/global_variable.py` 从 `base_config` 的 `map_id`、`data_id` 等生成数据路径；`utils/load_utils.generate_suffix(params)` 根据 `params` 生成实验后缀，用于保存模型/Excel/图。

### 4.2 主要参数（base_config + 代码内）

| 类别 | 变量/键 | 含义 | 典型值 |
|------|---------|------|--------|
| 数据 | `map_id`, `data_id` | 地图与数据集标识 | `"MarineMicro_MvsM_4"`, `"6"` |
| 序列 | `N`, `K` | 输入长度、预测步数 | 10, 5 |
| 模型选择 | `model_conf` | 主模型类型 | `"DecisionTransformer"` / `"TrajectoryTransformer"` |
| 模型名 | `model_name` | 用于路径与日志 | `decisionTransformer` / `sgTransformer` / `trajTransformer` |
| 模型开关 | `params["mdl_spatial_prior"]` | 预测时是否用距离空间先验 | bool |
| | `params["mdl_init_embedding_freeze"]` | 是否冻结 MDS 初始化的 embedding | bool |
| | `params["mdl_init_embedding_train"]` | 是否用 MDS 初始化但继续训练 | bool |
| | `params["mdl_attn_sim_bias"]` | 是否使用 SGTransformer 的相似度偏置 | bool |
| 训练 | `d_model`, `epochs`, `batch_size`, `lr` | 在 `train_*.py` 或 `prediction_RTS.py` 调用处写死/传参 | 如 128, 20, 32, 1e-4 |
| 环境 | `main.py` 内 `_MAP_CONFIG`, `_ENV_CONFIG`, `_ALG_CONFIG`, `_PATH_CONFIG` | 地图、步数、分辨率、路径等 | 见 main.py |

### 4.3 小结

- 超参数以 **单文件 Python 配置 + 少量调用处传参** 为主，无命令行或 YAML 统一入口，改实验需改 `base_config.py` 或相应训练脚本。

## 五、方法创新与学术定位

### 5.1 主要方法创新

1. **状态离散化与几何结构**  
   用 BK-Tree + 聚类将 RTS 原始状态映射为离散状态 ID，再通过 **自定义状态距离**（含空间与生命值）和 **DTW** 得到状态/轨迹间的几何关系，为后续“距离感知”建模打基础。

2. **距离诱导的 Transformer 表征**  
   - **MDS 初始化**：用状态距离矩阵做 MDS，初始化 TrajectoryTransformer 的 embedding 或 DT 的 state embedding，使表征空间与状态距离一致。  
   - **SGTransformer**：在注意力中引入 `sim_bias`（由状态距离等构造），用可学习标量缩放，使注意力受“状态相似度”引导。  
   - **空间先验**：自回归预测时用状态距离构造高斯先验，与 logits 相乘，偏向几何近邻状态。

3. **RTS 场景下的 Decision Transformer**  
   将 DT 用于离线 RTS 轨迹：状态=离散状态 ID，动作=离散动作表，RTG 由每步奖励反向累加；用 MDS 初始化 state embedding，在“状态+回报→动作”的框架下做策略表达与评估。

4. **适应度/状态景观可视化**  
   用轨迹 DTW 距离 + 游戏结果做 MDS + 插值得到 Fitness Landscape；用状态距离矩阵 + 状态价值做 State Landscape，用于分析轨迹质量与状态价值分布。

### 5.2 学术定位与可行性

- **定位**：介于 **RTS AI / 游戏 AI** 与 **离线强化学习 / 序列建模** 之间；强调“状态与轨迹的几何表示 + Transformer 预测/决策”，可与 Trajectory Transformer、Decision Transformer、以及 RTS 状态抽象、Fitness Landscape 等工作对话。  
- **可行性**：  
  - 数据链完整：从 PySC2 采集到 BK-Tree、距离矩阵、DTW、Fitness/State Landscape 再到 Transformer/DT 训练与评估，流程可复现。  
  - 创新点明确：距离诱导的初始化与注意力、RTS 专用状态距离与离散化、DT 在 RTS 离线数据上的应用，均可单独或组合写成方法章节与实验。  
- **可改进点**：  
  - 参数管理：引入 argparse 或 Hydra + YAML，便于消融与复现。  
  - 实验记录：对 `params` 与 `suffix` 做日志或持久化，便于对应到每次运行的模型与图表。  
  - 对比基线：与不用 MDS/SG/先验的 Vanilla Transformer、以及其它离线 RL 方法做对比，可加强说服力。
